# 14. 리스크, 컴플라이언스, 운영 주의사항

## 1. 투자/법적 주의

이 도구는 개인 학습과 모의투자 검증용이다. 추천 결과는 투자 자문이 아니며, 실제 투자 판단과 책임은 사용자에게 있다.

## 2. API/약관 주의

- 토스증권 Open API 이용약관을 확인한다.
- API Key는 거래 지시 권한을 가질 수 있으므로 비밀번호처럼 취급한다.
- 비공식 API나 내부 API를 사용하지 않는다.
- 공식 Open API만 사용한다.
- API 스펙은 변경될 수 있으므로 개발 전 최신 OpenAPI JSON을 확인한다.

## 3. 자동매매 리스크

자동매매는 편리하지만 다음 위험이 있다.

- 버그로 인한 반복 주문
- 급변장 손실 확대
- 지연 시세 기반 오판
- API 장애/네트워크 장애
- rate limit으로 인한 주문/조회 실패
- Secret 유출 시 계좌 피해
- 과도한 주문으로 인한 계정 제한 또는 법적 문제

## 4. v0.1 안전선

- 실주문 없음
- paper trading only
- AI 자동 주문 없음
- 고위험 종목 차단
- 로그 마스킹
- 테스트 필수

## 5. 운영 체크리스트

### 매일 시작 전

- `.env` 설정 확인
- API token 발급 확인
- 장 운영 시간 확인
- rate limit 오류 여부 확인
- 마지막 에러 로그 확인

### 장애 발생 시

- `logs/error.log` 확인
- requestId 확인
- API 응답 code 확인
- 마지막 정상 snapshot으로 fallback
- 반복 실패 시 API 호출 중단

### Secret 유출 의심 시

1. 앱 즉시 중지
2. `.env` 삭제 또는 키 제거
3. 토스증권에서 Client Secret 재발급
4. 로그 파일에서 노출 여부 확인
5. Git history에 올라갔는지 확인

## 6. 실주문 버전으로 넘어가기 전 필수 조건

- 최소 2~4주 paper trading 기록
- 최대 손실 제한 검증
- 주문 preview 기능
- 수동 승인 workflow
- kill switch
- 주문 취소/정정 상태 추적
- 소액 실거래 수동 테스트
- 이용약관/규정 검토
