# 01. 제품 요구사항 정의서

## 1. 제품명

가칭: `Toss AI Paper Trader`

## 2. 한 줄 설명

토스증권 Open API로 국내/미국 주식 데이터를 조회하고, AI가 분석 설명을 생성하며, 실제 돈을 쓰지 않는 모의투자로 전략을 검증하는 개인용 트레이딩 도구.

## 3. v0.1 범위

### 포함

- API 인증 설정
- 관심종목 관리
- 현재가/호가/체결/캔들/상하한가 조회
- 종목 기본정보/매수 유의사항 조회
- 환율/장 운영 시간 조회
- 계좌 목록/보유 주식/매수 가능 금액/판매 가능 수량/수수료 조회
- AI 추천 설명
- 모의 포트폴리오
- 모의 주문/모의 체결
- 추천/모의 주문/체결/에러 로그
- 테스트 결과 자동 리포트

### 제외

- 실제 주문 자동 실행
- 실시간 WebSocket 시세
- 다중 사용자 SaaS
- 외부 배포
- 리밸런싱 자동 주문
- 세금 신고용 확정 계산

## 4. 사용자 시나리오

### 시나리오 A — 관심종목 분석

1. 사용자가 관심종목에 `005930`, `AAPL`을 등록한다.
2. 앱이 종목 정보, 현재가, 캔들, 경고 정보를 조회한다.
3. rule-based score가 후보를 정렬한다.
4. AI가 왜 추천/관망/제외인지 자연어로 설명한다.
5. 사용자는 추천 결과를 보고 모의 주문을 생성한다.

### 시나리오 B — 보유 주식 기반 분석

1. 사용자가 토스증권 계좌를 연결한다.
2. 앱이 계좌 목록에서 `accountSeq`를 확인한다.
3. 앱이 보유 주식을 조회한다.
4. 보유 종목별 손익, 변동성, 경고 여부를 분석한다.
5. AI가 보유/축소/관망 후보를 설명한다.

### 시나리오 C — 모의투자 검증

1. 사용자가 추천 종목을 모의 매수한다.
2. 앱은 현재가 또는 최근 체결가를 기준으로 paper fill을 생성한다.
3. 이후 가격 갱신 시 평가손익을 계산한다.
4. 리포트에서 누적 수익률, 승률, MDD를 확인한다.

## 5. 추천 등급

- `BUY_CANDIDATE`: 매수 후보
- `WATCH`: 관망
- `HOLD`: 보유 유지
- `REDUCE`: 축소 후보
- `BLOCKED`: 매매 금지/제외

## 6. 필수 안전 요구사항

- 경고 종목은 기본적으로 `BLOCKED` 또는 강한 감점
- 장 운영 시간 외 실주문 금지
- 실주문은 v0.1에서 전면 비활성화
- AI 설명에는 확정 수익 표현 금지
- “반드시 오른다”, “안전하다”, “보장” 같은 문구 금지
- 모든 주문성 행위는 paper order로만 저장

## 7. 비기능 요구사항

- 초보 운영자가 이해할 수 있는 로그
- 에러 발생 시 requestId 저장
- API rate limit 감지 및 backoff
- Secret masking
- 테스트 자동화
- 데이터 백업 쉬움: SQLite 단일 파일
