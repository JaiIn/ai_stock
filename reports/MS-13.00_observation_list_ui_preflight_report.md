# MS-13.00 Observation List UI Preflight Report

## Purpose

MS-12.00 recommendation list model preflight, MS-12.01 recommendation list
fixtures/evaluators, and MS-12.02 recommendation list fixture hardening outputs
were reused to add an observation-only UI preflight row/view model contract.
This stage prepares future Streamlit display shapes without importing
Streamlit, rendering UI, modifying app code, creating an actual recommendation
list, ranking candidates, or producing trade directives.

## UI Guardrails

- Streamlit import: forbidden.
- `app/streamlit_app.py` modification: forbidden.
- Button/callback/session state: forbidden.
- API refresh, OAuth login, credential input, and accountSeq input: forbidden.
- Order/trade/account/holdings/balance/fills UI: forbidden.
- `status_badge` cannot be buy/sell/hold.
- `score_snapshot_label` cannot be recommendation, ranking, or action.
- `display_bucket` cannot be ranking, priority, or order.
- `usability_label` cannot be a ranking flag.
- Required disclaimer labels must include observation-only and no-trade meaning.
- Forbidden output fields and assignment keywords must not be copied to rows.
- Required external capability flags must remain false.
