# MS-06.01 Read-only Snapshot Ingestion Service Report

## 수행 목적

지금까지 검증한 read-only API 결과를 이후 로컬 저장소와 연결하기 위한

반환한 이미 파싱된 model object만 `LocalDataPersistenceService`로 저장합니다.

## service 설계 요약

`ReadOnlySnapshotIngestionService`를 추가했습니다. 하나의 read-only snapshot
request를 받아 아래 데이터를 저장합니다.

- `StockInfo`
- `PriceSnapshot`
- `CandlePage` 또는 `Candle`
- `ExchangeRate`

저장은 새 schema나 새 repository 없이 기존 `LocalDataPersistenceService`를
재사용합니다.

## dependency injection 구조 요약

서비스는 다음 provider Protocol을 생성자에서 주입받습니다.

- `StockInfoSnapshotProvider`
- `PriceSnapshotProvider`
- `CandleSnapshotProvider`
- `ExchangeRateSnapshotProvider`

각 provider는 이미 파싱된 domain model만 반환합니다. 서비스는 Toss client,
OAuth provider, `httpx`, request context, token provider, 환경설정을 import하지
않습니다.

## 저장 대상 요약

요청 예시는 다음 값을 기준으로 검증했습니다.

- symbol: `005930`
- exchange rate pair: `USD/KRW`
- candle interval: `1d`
- candle count: `1`
- adjusted: `true`

결과 객체 `ReadOnlySnapshotIngestionResult`는 다음 안전 필드를 반환합니다.

- requested symbol
- saved stock count
- saved price snapshot count
- saved candle count
- saved exchange rate count
- stock warnings deferred 여부
- skipped stock warnings count
- success 여부
- safe message
- `actual_network_call_performed=false`

## StockWarnings persistence deferred 사유

Stock Warnings live smoke 성공 기록은 유지합니다. 다만 현재 로컬 DB schema와
repository에는 stock warnings 전용 저장소가 없으므로 이번 단계에서는 억지로
schema migration이나 repository를 추가하지 않았습니다. StockWarnings persistence
연계는 별도 Micro Stage로 deferred 처리했습니다.

## validation 요약

`ReadOnlySnapshotIngestionRequest` validation:

- symbol required
- symbol 허용 문자: 영문, 숫자, `.`, `-`
- candle interval: `1m` 또는 `1d`
- candle count: `1~200`
- adjusted: boolean only
- live mode flag 거부
- real order / order send flag 거부
- account scope / accountSeq 입력 거부
- exchange-rate currency pair required

## in-memory SQLite 테스트 요약

테스트는 `create_connection(":memory:")`만 사용합니다. 실제 DB 파일을 만들지
않고 기존 schema initialization과 repository를 in-memory SQLite에만 적용합니다.
