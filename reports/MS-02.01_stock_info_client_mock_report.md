# MS-02.01 Stock Info Client Mock 보고서

## 수행 범위

- `TossStockInfoClient` request-only 인터페이스 추가
- `getStocks`용 `GET /api/v1/stocks` 요청 정의
- `getStockWarnings`용 `GET /api/v1/stocks/{symbol}/warnings` 요청 정의
- 문서화된 최소 종목 기본 정보 및 경고 정보 모델 추가
- fake `result` 배열 기반 응답 parsing 추가
- 실패형 `MockTransport` 기반 무전송 테스트 추가

## 안전성 확인

- 실제 Toss API 및 OAuth 호출 없음
- HTTP 전송 메서드 또는 계좌/자산/주문 API 구현 없음
- 두 stock-info 요청 모두 계좌 헤더를 요구하거나 생성하지 않음
- Authorization 원문과 raw headers를 예외 및 보고서에 저장하지 않음
- 테스트 값은 mock/fake 값만 사용

## OpenAPI 재검증 필요 사항

- 실제 응답의 `result` wrapper 내부 구조는 live 호출 없이 확인하지 않았으므로 재검증 필요
- 문서에 없는 `description` 필드는 경고 모델에 임의 추가하지 않음
- `koreanMarketDetail`의 내부 필드는 범위에서 제외했으며 추후 schema 재검증 필요
