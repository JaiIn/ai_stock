# MS-06.04 Live Read-only Snapshot Ingestion Smoke Report

## 수행 목적

MS-06.03에서 고정한 단일 5-call plan에 따라 live read-only API 결과를 공식
model로 즉시 파싱하고, `ReadOnlySnapshotIngestionService` provider DI 흐름을
통해 in-memory SQLite에 저장·재조회했습니다.

저장 대상은 `StockInfo`, `PriceSnapshot`, `Candle/CandlePage`,
`ExchangeRate`로 제한했습니다. StockWarnings는 제외하고 deferred 상태를
유지했습니다.

## strict single-pass 실행 구조

실행 경로는 strict call ledger로 method, path, query, 순서를 검증하고 여섯 번째
호출이나 순서·query 불일치를 차단합니다. 자동 재시도와 retry loop는 없습니다.
각 업무 GET 전에 Live API Safety Gate를 평가합니다.

## 실제 호출 범위

승인된 script를 정확히 1회 실행했습니다.

1. `POST /oauth2/token`: 1회
2. `GET /api/v1/stocks?symbols=005930`: 1회
3. `GET /api/v1/prices?symbols=005930`: 1회
4. `GET /api/v1/candles?symbol=005930&interval=1d&count=1&adjusted=true`: 1회
5. `GET /api/v1/exchange-rate?baseCurrency=USD&quoteCurrency=KRW`: 1회

- OAuth token endpoint call count: `1`
