## Mock-only policy model 요약

`MockRecommendationPolicy`, `MockRecommendationInput`,
`MockRecommendationResult`, `MockRecommendationValidationResult`를 frozen
dataclass로 정의했다. `MockRecommendationLabel`은 다음 비지시적 label만
허용한다.

- `observation_only`
- `risk_review`
- `insufficient_data`
- `neutral_review`
- `not_evaluated`

직접적인 투자 행동을 뜻하는 영문·한글 label은 금지 목록으로만 관리하며
builder가 생성하지 않는다. 금지 label과 금지 문구는 validation negative
fixture로만 사용된다.

입력은 caller-supplied snapshot summary의 symbol, 네 가지 데이터 존재
flag, completeness flags, risk flags, observation notes만 받는다. 이
모듈은 DB, `local_snapshot_latest_read_model`, API refresh 또는 다른
데이터 소스를 직접 호출하지 않는다. caller 문자열 원문은 결과에
반영하지 않고 안전한 개수와 고정 설명만 사용한다.

## Deterministic pure no-I/O 확인

모듈 의존성은 `dataclasses`, `enum`, MS-08.01 safety contract뿐이다.
파일, DB, 환경변수, 네트워크, Streamlit, Toss client, OpenAI/LLM,
현재 시각, random을 사용하지 않는다. 같은 입력과 정책은 항상 같은
frozen 결과를 반환한다.

## 결과 규칙

- 필수 snapshot summary가 부족하거나 completeness flag가 있으면
  `insufficient_data`
- 데이터가 완전하고 risk flag가 있으면 `risk_review`
- 데이터가 완전하고 risk flag가 없으면 `observation_only`
- 입력 또는 정책 validation이 실패하면 `not_evaluated`
- 모든 결과에 MS-08.01 required disclaimer 6개 포함
- 모든 결과에서 `is_investment_advice=false`
- 모든 결과에서 `action_allowed=false`
- 모든 결과에서 `real_trade_allowed=false`
- 같은 입력과 정책은 항상 같은 결과

설명은 MS-08.01의 allowed explanation language 8개 중 하나를 prefix로
사용한다. 금지 recommendation language 10개가 결과에 포함되면 validation
FAIL이다. caller가 전달한 금지 문구나 잘못된 symbol은 echo하지 않고
`invalid_input`과 고정된 safe explanation으로 fail closed 처리한다.
