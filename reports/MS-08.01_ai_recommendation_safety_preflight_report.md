## Safety preflight 정책 요약

`RecommendationSafetyPreflightPolicy`는 frozen dataclass이며 기본값이 모두
deny-by-default인 순수 정책 객체다. 모듈은 표준 라이브러리 `dataclasses`
외 의존성이 없고 파일, DB, 환경변수, 네트워크, Streamlit, Toss client,
OpenAI/LLM, 현재 시각을 사용하지 않는다. 정책 생성, 검증, 요약 함수는
입력에 대해 결정적인 결과만 반환하며 추천 결과를 산출하지 않는다.

| 정책 항목 | MS-08.01 값 |
| --- | --- |
| stage_name | `MS-08.01` |
| mode | `safety_preflight` |
| real recommendation allowed | `false` |
| mock recommendation allowed | `false` |
| LLM call allowed | `false` |
| external AI API allowed | `false` |
| OpenAI API KEY required | `false` |
| Toss API KEY required | `false` |
| Toss SECRET KEY required | `false` |
| OAuth allowed | `false` |
| live market API allowed | `false` |
| account API allowed | `false` |
| order API allowed | `false` |
| accountSeq required | `false` |
| DB write allowed | `false` |
| Streamlit UI change allowed | `false` |
| real trade allowed | `false` |

따라서 실제 추천과 mock 추천 모두 이 단계에서 허용되지 않는다. OpenAI와
Toss credential은 불필요하며, OAuth, live market, account, order,
accountSeq, DB write, Streamlit UI 변경 및 실거래도 허용되지 않는다.

## Required disclaimer

다음 의미를 고정된 tuple 계약으로 정의했다.

- 이 기능은 투자 조언이 아닙니다.
- 실제 매수/매도/보유 지시가 아닙니다.
- 실거래 전 별도 판단이 필요합니다.
- 모델/룰 기반 출력은 오류가 있을 수 있습니다.
- 손실 가능성이 있습니다.
- 이 프로젝트는 local-only 실험 도구입니다.

## 언어 정책

금지 recommendation language에는 `지금 사라`, `반드시 매수`,
`무조건 상승`, `확정 수익`, `손실 없음`, `원금 보장`, `몰빵`,
`실주문 실행`, `계좌에서 매수`, `자동 주문`을 포함한다. 이는 실제 추천
문구가 아니라 향후 출력에서 반드시 거부해야 할 금지어 계약이다.

허용 explanation language에는 `관찰 포인트`, `리스크 요인`, `확인 필요`,
`시나리오`, `모의 판단`, `데이터 기반 참고`, `투자 조언 아님`,
`실거래 금지`를 포함한다. 허용 목록은 설명 범위를 제한하기 위한
정책이며 매수·매도·보유 추천을 생성하지 않는다.

## MS-08.02 mock-only 후보 범위

`MockOnlyRecommendationPlan`으로 다음 단계 후보를 별도 정의했으며
MS-08.01에서는 실행하거나 활성화하지 않았다.

- mock-only 입력
- local snapshot read model 기반 DTO 사용 가능
- 실제 API refresh 없음
- 실제 AI/LLM 호출 없음
- rule-based 또는 deterministic placeholder 가능
- 투자 자문 금지
- 실주문 금지
- explanation 중심
- 직접 매수·매도 지시 형태의 recommendation label 금지
- 출력 후보는 관찰 포인트, 리스크 요인, 데이터 완전성,
  시나리오 설명, required disclaimer로 제한
