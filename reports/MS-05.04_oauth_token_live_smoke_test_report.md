## OAuth token provider / smoke test 구조 요약

- Endpoint는 `/oauth2/token` 상수로 고정했습니다.
- `application/x-www-form-urlencoded` form으로 `grant_type`, `client_id`,
  `client_secret`만 전송합니다.
- Response의 `access_token`, `token_type`, `expires_in`을
  `OAuthTokenResponse`로 복원합니다.
- Provider는 주입된 HTTP client만 사용하며 token을 저장하지 않습니다.
- Script에서만 실제 `httpx.Client`를 생성하며 다른 Toss endpoint 경로가 없습니다.

## credential 처리 방식 요약

- Credential은 `.env.local`의 `TOSS_CLIENT_ID`, `TOSS_CLIENT_SECRET`에서만 읽습니다.
- `.env.local`이 없거나 credential이 없으면 live smoke test를 차단합니다.
- Client ID/Secret, request body, error response body 원문을 출력하지 않습니다.
- accountSeq와 기존 Access Token은 요청하거나 사용하지 않습니다.

## .env.local Git 제외 확인 결과

- `.gitignore`의 `.env.local` 규칙을 `git check-ignore -v .env.local`로 확인했습니다.
- 현재 `.env.local`은 존재하지 않습니다.

## token masking / safe dump 처리 요약

- `OAuthTokenResponse`는 Access Token을 `repr`에서 제외합니다.
- Script는 `safe_dict()`만 출력하며 token 원문은 출력하지 않습니다.
- Token은 메모리에만 존재하고 파일 저장 경로가 없습니다.

## Live API Safety Gate 연계 요약

- OAuth smoke test 전 `ALLOW_LIVE_API=true`를 요구합니다.
- `ALLOW_REAL_ORDER=true`이면 차단합니다.
- `DRY_RUN_ONLY=true`를 요구합니다.
- OAuth endpoint 예외는 업무 API allowlist를 확장하지 않습니다.
