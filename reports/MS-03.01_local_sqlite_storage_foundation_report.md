# MS-03.01 로컬 SQLite 저장소 기반 보고서

## 수행 범위

- 표준 라이브러리 `sqlite3` 기반 connection helper 추가
- 기본 로컬 경로 `data/ai_stock.sqlite3` 정의
- schema version 1과 승인된 데이터 테이블 초기화 추가
- 기존 StockInfo, PriceSnapshot, Candle, ExchangeRate 모델용 repository 추가
- in-memory SQLite 기반 CRUD 및 무전송 테스트 추가

## Schema 요약

- `schema_version`: schema version과 적용 시각
- `stocks`: 종목 기본 정보
- `price_snapshots`: 종목별 시세 snapshot
- `candles`: 종목·interval별 OHLCV
- `exchange_rates`: 통화쌍별 참고 환율

계좌, 자산, 주문, 체결, 잔고, 포트폴리오, 인증정보 저장 테이블은 생성하지 않는다.

## 저장 정책

- 가격, OHLCV, 환율은 SQLite `TEXT`로 저장하고 Python에서는 `Decimal`로 복원
- timestamp와 date_time은 현재 ISO 형태 문자열로 저장
- DB connection은 caller가 소유하며 repository는 전달받은 connection을 사용
- schema 초기화는 idempotent하며 version 1을 한 번만 기록

## 로컬 전용 및 안전성 확인

- 테스트 DB는 `:memory:`에만 생성하며 로컬 파일을 남기지 않음
- repository 모듈은 네트워크 client에 의존하지 않음
- 기존 `pyproject.toml`의 SQLAlchemy 선언은 변경하지 않았으며 이번 구현에서 사용하지 않음
- socket 차단 상태에서 조회 작업이 정상 동작함을 테스트
- API Key, token, secret, accountSeq 또는 계좌번호를 저장하는 column 없음
- `.gitignore`가 `data/ai_stock.sqlite3`, `*.sqlite`, `*.sqlite3`, `*.db`를 제외함

## 추후 보완 사항

- timestamp/date_time의 timezone 및 정규화 규칙 확정 필요
- schema 변경이 필요해지면 사용자 승인 후 migration 전략 별도 수립
- snapshot 중복 방지 key와 보존 기간은 후속 단계에서 확정 필요
- 로컬 DB 백업·복구 절차는 후속 운영 단계에서 정의 필요

## 테스트 환경 확인

- 최초 시스템 TEMP 파일 테스트는 Windows sandbox 쓰기 권한으로 실패함
- 요구사항에서 허용한 `:memory:` SQLite로 전환한 뒤 전체 테스트 통과
- workspace와 `data/` 아래에는 테스트 DB 파일을 생성하지 않음
