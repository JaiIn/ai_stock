## Read-only live smoke test 구조

- `TossReadOnlySmokeClient.get_exchange_rate_once()`만 제공합니다.
- endpoint 상수는 `/api/v1/exchange-rate`로 고정했습니다.
- 명시적 `read_only_live_smoke_allowed=True`가 없으면 transport 호출 전에 차단합니다.
- 응답은 기존 `TossMarketInfoClient.parse_exchange_rate_response()`로 파싱합니다.
- 결과 객체는 HTTP status와 parsed `ExchangeRate`만 메모리에 보유합니다.

## OAuth token provider 재사용

- MS-05.04의 `TossOAuthTokenProvider`를 변경 없이 재사용합니다.
- token은 메모리 객체로만 전달하며 파일에 저장하지 않습니다.

## Live API Safety Gate 연계

- method: `GET`
- path: `/api/v1/exchange-rate`
- category: `market_info`
- requires auth: `true`
- requires accountSeq: `false`
- required settings: `ALLOW_LIVE_API=true`, `ALLOW_REAL_ORDER=false`, `DRY_RUN_ONLY=true`
- safety decision이 allowed가 아니면 HTTP transport 전에 중단합니다.

## 허용된 실제 API 호출 범위

- `POST /oauth2/token` 최대 1회
- `GET /api/v1/exchange-rate` 최대 1회
- 그 외 업무, 시세, 계좌, 자산, 잔고, 체결, 주문 endpoint는 허용하지 않습니다.

## Live smoke test 성공/실패 요약

- 실패
- smoke script는 민감정보를 노출하지 않는 안전 실패 메시지만 반환했습니다.
- 기존 실패 당시에는 OAuth 단계와 업무 API 단계 및 HTTP status가 보존되지 않아
  정확한 실패 지점을 확인할 수 없었습니다.
- 이번 보강 이후 후속 실행에서는 phase, HTTP status, safe error type/message를
  확인할 수 있습니다.

## HTTP status code

- 기존 live 실패의 status: 확인 불가
- 후속 실행에서는 가능한 경우 OAuth 또는 환율 endpoint HTTP status를 보존합니다.
- network timeout처럼 status가 없으면 `null`로 유지합니다.
- 응답 원문 또는 request 원문을 재현하기 위한 추가 호출은 수행하지 않았습니다.

## Parsed response schema 요약

- 예상 공식 필드: `baseCurrency`, `quoteCurrency`, `rate`, `validFrom`, `validUntil`
- live 실행이 parsing 성공까지 도달하지 않아 parsed field 존재 여부는 확인하지 못했습니다.
- 실제 response 전체와 `rate` 값은 출력하거나 저장하지 않았습니다.
