- API version: `1.1.5`
- Endpoint: `GET /api/v1/candles`
- operationId: `getCandles`
- Security: OAuth2 Client Credentials
- accountSeq required: false
- Read-only: true
- Rate Limits Group: `MARKET_DATA`

## Request query schema

- `symbol`: required
- Symbol pattern: `^[A-Za-z0-9.\-]+$`
- Examples: `005930`, `AAPL`
- `interval`: required
- `interval` allowed values: `1m`, `1d`
- `count`: optional, default `100`, minimum `1`, maximum `200`
- `before`: optional ISO 8601 date-time, exclusive pagination upper bound
- `adjusted`: optional boolean, default `true`
- Single-symbol request: supported
- Next proposed live candidate:
  `GET /api/v1/candles?symbol=005930&interval=1d&count=1&adjusted=true`
- Live execution in MS-05.14: not performed

## Response 200 schema

The root is the standard `{ "result": { ... } }` envelope.
Response `result` is `CandlePageResponse`.

- `result.candles`: required array
- `result.nextBefore`: optional/nullable date-time pagination cursor
- Each candle requires:
  - `timestamp`: date-time string
  - `openPrice`: decimal string
  - `highPrice`: decimal string
  - `lowPrice`: decimal string
  - `closePrice`: decimal string
  - `volume`: decimal string
  - `currency`: required currency string
- OHLCV fields are parsed as finite `Decimal`
- Existing storage-compatible `Candle` and `CandlePage` objects remain
  compatible while the official parser now requires `currency`.
- `Candle.from_official_mapping` 추가
- `CandlePage.from_official_mapping` 추가

## Error schema

`CandleError` safe parser를 추가했습니다. The Candles error parser retains only
documented safe metadata:

- `error.requestId`
- `error.code`
- `error.message`
- optional `error.data.field`
- optional `error.data.allowedValues`
- optional `error.data.constraint.min`
- optional `error.data.constraint.max`

Covered response categories:

- 400 `invalid-request`
- 404 common not-found response
- 429 `rate-limit-exceeded`
- 500 `internal-error`

Raw request bodies, raw response bodies, authorization headers, credentials,
and tokens are not retained by the model.

## Live API Safety Gate

- method: `GET`
- path: `/api/v1/candles`
- category: `market-data`
- requires auth: true
- requires accountSeq: false
- read-only allowlist: yes
- order/write/mutation category: no
- `requires_account_seq=true`: blocked

No change to the production safety allowlist was required. Tests now explicitly
cover the Candles account-free decision and fail-closed account-scoped metadata.
