# MS-10.03 Feature Extraction Fixture Hardening Report

## Purpose

MS-10.03 adds a deterministic feature extraction fixture/evaluator hardening
layer on top of the MS-10.02 feature extraction preflight contract. The stage
expands expected-vs-actual checks for extraction readiness, review, blocked,
missing-data, sanitized, and empty-input cases. It does not implement scoring,
ranking, recommendation, target price, expected return, profit probability,
UI integration, storage, file loading, DB access, Toss API, OAuth, OpenAI/LLM,
accountSeq, or order/account features.

## Feature Extraction Fixture Hardening Summary

The new hardening layer defines immutable extraction fixture records and
evaluation results. It reuses MS-10.02 extracted feature sets and compares
their extraction statuses, ready counts, review counts, future-scoring
usability counts, missing/blocked feature keywords, blocked reasons, warnings,
diagnostics, required false flags, and forbidden keyword absence.

## Evaluation Policy

- `extraction_basic_ready`: validates two `extraction_ready` outputs.
- `extraction_mixed_review`: validates one ready output and two invalid
  candidate review outputs.
- `extraction_duplicates_blocked`: validates duplicate candidate blocking.
- `extraction_disabled_blocked`: validates disabled candidate blocking.
- `extraction_missing_data_blocked`: validates missing data extraction blocking.
- `extraction_forbidden_field_sanitized`: validates diagnostics-only
  forbidden-field sanitization with no raw forbidden value copies.
- `extraction_empty_input`: validates safe empty input extraction state.
- `extraction_all_fixture_matrix`: validates all 11 extraction outputs from
  the six MS-09.04 fixture groups.

## Forbidden Output Sanitization

access token assignment, authorization/bearer assignment, API/secret/client

recommendation, action, buy, sell, hold, target price, expected return, and
profit probability assignment forms. Sanitized field names may appear only in
diagnostics such as `forbidden_field_detected:accountSeq`.
