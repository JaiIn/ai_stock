# MS-05.09 Stock Info Read-only Schema Preflight Report

## 수행 내용

실제 API 호출 없이 공식 Toss OpenAPI JSON을 일회성으로 조회하여 Stock Info
read-only endpoint의 request, response, error schema를 재검증했습니다. 기존
request builder, model/parser, fake transport 테스트와 Live API Safety Gate를
공식 schema에 맞게 정렬했습니다.

## Stock Info endpoint schema

### getStocks

- method/path: `GET /api/v1/stocks`
- operationId: `getStocks`
- security: OAuth2 client credentials
- accountSeq required: false
- read-only: true
- `symbols`: required query
- 개수: 1~200개
- 형식: 쉼표 구분, 개별 symbol은 영문 대/소문자·숫자·`.`·`-`
- 단일 symbol 요청: 가능
- responses: 200, 400, 401, 403, 429, 500

### getStockWarnings

- method/path: `GET /api/v1/stocks/{symbol}/warnings`
- operationId: `getStockWarnings`
- security: OAuth2 client credentials
- accountSeq required: false
- read-only: true
- path `symbol`: required
- 형식: 영문 대/소문자·숫자·`.`·`-`
- responses: 200, 401, 403, 404, 429, 500
- 종목이 없으면 404 `stock-not-found`
- 활성 warning이 없으면 200과 빈 `result` 배열

## response schema 정렬

StockInfo 공식 required 필드:

- `symbol`
- `name`
- `englishName`
- `isinCode`
- `market`
- `securityType`
- `isCommonShare`
- `status`
- `currency`
- `sharesOutstanding`

optional 또는 nullable 필드:

- `listDate`
- `delistDate`
- `leverageFactor`
- `koreanMarketDetail`

`sharesOutstanding`와 `leverageFactor`는 Decimal로 처리합니다.
국내시장 상세 객체는 liquidation/NXT 지원/KRX·NXT 거래정지 상태를 안전한
중첩 모델로 parsing합니다. 기존 저장 계층의 최소 생성자 호환성은 유지합니다.

StockWarning은 required `warningType`과 optional `exchange`, `startDate`,
`endDate`를 parsing하며 unknown warning code를 문자열로 허용합니다.

## error schema 정렬

raw response body를 저장하지 않고 아래 안전 필드만 추출합니다.

- `error.requestId`
- `error.code`
- `error.message`
- `error.data.field`
- `error.data.allowedValues`
- `error.data.constraint.min`
- `error.data.constraint.max`

검증 대상:

- 400 `invalid-request`
- 404 `stock-not-found`
- 429 `rate-limit-exceeded`
- 500 `internal-error`

## Live API Safety Gate 연계

- `GET /api/v1/stocks`: read-only allowlist 유지
- `GET /api/v1/stocks/{symbol}/warnings`: read-only allowlist 유지
- requires_auth: true
- requires_account_seq: false
- category: stock-info
- order/write/mutation 아님
- 같은 path라도 requires_account_seq=true metadata는 차단
- 주문 및 mutation denylist는 변경 없음
