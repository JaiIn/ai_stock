# MS-08.03 Recommendation Explanation UI Preflight Report

## Purpose

MS-08.03 defines the safe explanation UI contract that a future Streamlit panel
can use to display an already prepared MS-08.02 mock-only recommendation result.
This stage does not connect to Streamlit and does not generate a new
recommendation. It only builds and validates a deterministic display ViewModel
from a caller-supplied `MockRecommendationResult`.

## Deterministic Pure No-I/O Confirmation

The module uses only `dataclasses`, `enum`, and existing MS-08.01/MS-08.02
recommendation contracts. It does not import or call file, DB, environment,
network, Streamlit, Toss client, OpenAI/LLM, clock, random, account, or order
dependencies. It does not read `data/local/ai_stock.sqlite3`, `.env.local`, or
any DB row.

The builder accepts only an already prepared `MockRecommendationResult`. It
does not call the local snapshot read model, does not refresh API data, does not
read storage, and does not recalculate the mock recommendation.

## Validation Coverage

Validation passes for the default policy and default ViewModel. Validation
fails when:

- a forbidden section is included
- forbidden recommendation language is included
- a buy/sell/hold direct label is used
- a required disclaimer is missing
- `action_allowed=true`
- `is_investment_advice=true`
- `real_trade_allowed=true`
- `order_button_allowed=true`
- `credential_input_allowed=true`
- `account_seq_input_allowed=true`
- `live_refresh_allowed=true`
- `oauth_allowed=true`

Required disclaimers are copied from MS-08.01 and included in the ViewModel.
Allowed explanation language and safe labels are reused from MS-08.01/MS-08.02.
