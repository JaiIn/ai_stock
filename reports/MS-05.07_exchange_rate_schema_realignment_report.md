## Exchange-rate query schema

- `baseCurrency`: required, Currency enum
- `quoteCurrency`: required, Currency enum
- `dateTime`: optional, ISO 8601 date-time
- Currency enum: `KRW`, `USD`
- 동일 통화쌍은 공식 400 example 대상이며 local request 생성 전에 차단합니다.
- 지원하지 않는 통화도 local request 생성 전에 차단합니다.

MS-05.02의 “optional dateTime only” 기록은 당시 alignment 이력으로 유지하되,
MS-05.07 공식 schema 재확인 결과 잘못된 가정으로 정정했습니다.

## Response 200 schema

공식 `result` required fields:

- `baseCurrency`
- `quoteCurrency`
- `rate`
- `midRate`
- `basisPoint`
- `rateChangeType`: `UP`, `EQUAL`, `DOWN`
- `validFrom`
- `validUntil`

`rate`, `midRate`, `basisPoint`는 `Decimal`로 파싱합니다. 기존 저장 계층 호환을
위해 `exchange_rate` field와 `rate` property를 유지했습니다. 신규 필드와 validity
필드는 기존 직접 생성 경로에서는 optional이지만 공식 response parser에서는
required입니다. `rateChangeType`은 `UP`, `EQUAL`, `DOWN`으로 검증합니다.

## Error schema

- 공통 envelope: `error`
- required: `requestId`, `code`, `message`
- optional data:
  - `field`
  - `allowedValues`
- 400:
  - unsupported currency
  - same currency
  - code `invalid-request`
- 404: `exchange-rate-not-found`
- 429: `rate-limit-exceeded`
- 500: `internal-error`

`ExchangeRateError`는 위 안전 metadata만 보존하며 raw response body는 저장하지
않습니다.

## MS-05.06 HTTP 400 원인 분석

- OAuth token 단계는 통과했습니다.
- `readonly_exchange_rate` phase에서 HTTP 400이 발생했습니다.
- 당시 smoke request는 `baseCurrency`와 `quoteCurrency` query를 보내지 않았습니다.
- 공식 API version 1.1.5에서는 두 query가 required입니다.
- 따라서 required currency query 누락이 HTTP 400 `invalid-request`의 원인일
  가능성이 높습니다.
- 다음 별도 승인 retry는 `baseCurrency=USD`, `quoteCurrency=KRW`를 명시하도록
  준비했습니다.
- 이번 단계에서는 live retry를 수행하지 않았습니다.
