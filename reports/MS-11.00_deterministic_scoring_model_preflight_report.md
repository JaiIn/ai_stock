# MS-11.00 Deterministic Scoring Model Preflight Report

## Purpose

MS-11.00 adds a pure no-I/O deterministic scoring preflight contract on top of
MS-10.02 extracted feature sets and MS-10.03 extraction fixture/evaluator
coverage. The stage defines only a safe data-quality scoring policy and model
shape for future scoring. It does not implement actual recommendation,
ranking, buy/sell/hold judgment, target price, expected return, profit
probability, UI integration, storage, file loading, DB access, Toss API, OAuth,
OpenAI/LLM, accountSeq, or order/account features.

## Deterministic Scoring Model Preflight Summary

The new preflight layer converts already in-memory extracted feature sets into
immutable scoring inputs, data-quality score components, and scoring preflight
results. Scores are deterministic, bounded by `score_scale`, and based only on
data completeness, quality reliability, review state, extraction readiness, and
sanitization state. They are not investment attractiveness, recommendation,
ranking, or trade action outputs.

## Scoring Preflight Policy

- Allowed sources: `in_memory_extracted_feature_set`,
  `watchlist_fixture_extraction`, `all_fixture_extraction`, and
  `feature_extraction_fixture_evaluation`.
- Forbidden sources include file paths, feature files, watchlist files,
  database/sqlite/table sources, Toss API, OAuth, OpenAI/LLM, account,
  holdings, order, balance, fills, raw API response, raw DB row, credentials,
  tokens, and accountSeq.
- Allowed score components are data-quality only:
  `data_completeness`, `quality_reliability`, `review_penalty`,
  `extraction_readiness`, and `sanitization_penalty`.
- Forbidden component/output names include trade labels, recommendation/ranking
  labels, target price, expected return, profit probability, position/order
  fields, account fields, credentials, tokens, raw request/response, DB rows,
  file paths, env paths, and `.env.local`.
- Required flags are inherited from MS-10 and remain false.

## Scoring Policies

- Extraction ready: maps to `score_ready` when usable for future scoring.
- Duplicate: maps to `score_duplicate_candidate`, review/blocked, no future
  ranking usability.
- Disabled: maps to `score_disabled_candidate`, review/blocked, no future
  ranking usability.
- Missing data: maps to `score_missing_data`, no future ranking usability.
- Invalid candidate: maps to `score_invalid_candidate`.
- Forbidden field sanitized: maps to `score_forbidden_field_sanitized`;
  forbidden fields remain diagnostics-only and raw forbidden values are not
  copied.
- Empty input: maps to `score_empty_input`.

## Fixture Scoring Coverage

- Fixture-based scoring is provided for each MS-09.04 in-memory fixture through
  MS-10.02 extraction.
- All-fixture scoring covers 11 extracted feature sets across the six MS-09.04
  fixture groups.
- MS-10.03 extraction fixture records can also be scored, including the
  all-fixture matrix.

## Total Score Boundary

`total_score` is deterministic and bounded by `score_scale=100`. It is computed
only from data completeness, quality reliability, review state, extraction
readiness, and sanitization state. It is not a recommendation, ranking, action,
target price, expected return, or profit probability.
