## Selector 설계

- default DB path: `data/local/ai_stock.sqlite3`
- default symbol: `005930`
- default exchange pair: `USD/KRW`
- UI 입력: symbol, base currency, quote currency
- symbol은 trim하며 빈 값은 `005930`으로 복원
- symbol은 화면 표시와 parameterized local query에만 사용하는 1~32자 안전 문자열
- currency code는 trim 후 대문자화하고 ASCII 영문 3자리만 허용
- valid symbol/pair만 `local_snapshot_latest_read_model` 호출에 전달
- invalid selector는 DB open 전에 `invalid_selector` safe warning 반환
- invalid selector 원문은 오류 메시지에 반영하지 않음

## app과 helper 변경

`app/streamlit_app.py`는 기존 title과 safe sections를 유지하면서 세 개의
`text_input`을 추가했습니다. credential, API key, secret, token, accountSeq 입력은
없습니다. API/OAuth/live refresh, DB write, 주문, 계좌, AI 추천 버튼도 없습니다.

presentation helper에는 immutable `DashboardSelection`과 순수
`validate_dashboard_selection`을 추가했습니다. 유효한 selector만 기존 read model
경계로 전달하며, validation failure에서는 SQLite connection을 열지 않습니다.

## Read-only와 표시 정책

- data source: `local_snapshot_latest_read_model`
- DB access: 기존 SQLite URI `mode=ro`, `PRAGMA query_only=ON`
- DB write, schema initialize, migration 경로: 없음
- DB 파일 존재 시 valid selector로 read-only 조회
- DB 파일 부재 시 생성 없이 `database_not_found` safe warning
- symbol 부재 또는 component 일부 부재 시 기존 warning/completeness 표시
- displayed safe sections, source counts, file metadata, diagnostics 유지
- DB row 전문, raw response/request body, credential/token/Auth header 표시 없음
- `actual_db_file_modified_this_stage=false`

유지한 safe sections:

- dashboard title, data health, safety status
- data source, DB path, selected symbol/pair
- StockInfo, PriceSnapshot, Candle, ExchangeRate
- completeness flags, source counts, read-only diagnostics, DB metadata

금지 UI actions:

- API/OAuth/live refresh
- DB write, migration, schema initialize
- credential 또는 accountSeq 입력
- 계좌/자산/잔고/체결 조회
- AI 추천, 주문 또는 실거래 action

금지 sensitive fields:

- DB raw row와 raw request/response body
- API key, Client ID/Secret, Access Token, Authorization header
- accountSeq와 실제 계좌/자산/잔고/체결 정보
