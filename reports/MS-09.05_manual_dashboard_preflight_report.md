# MS-09.05 Manual Dashboard Preflight Report

## Purpose

MS-09.05 defines a pure no-I/O dashboard preflight view model contract on top of
the MS-09.01 candidate input contract, MS-09.02 watchlist data model, MS-09.03
manual/local watchlist source adapter, and MS-09.04 watchlist source fixtures.
It prepares display-safe rows, counts, warnings, diagnostics, safety badges, and
next-action hints before any Streamlit UI integration, dashboard selector,
recommendation, scoring, persistence, API, OAuth, accountSeq, DB, file, account,
order, balance, asset, or fill behavior is introduced.

## Dashboard Preflight Summary

The preflight contract converts already in-memory MS-09.03 source results and
MS-09.04 fixtures into a dashboard-ready view model. It is deterministic,
side-effect free, and intentionally does not render Streamlit UI or create a
dashboard selector.

These terms are policy/test/documentation terms only and are not generated as
dashboard badges, action labels, recommendation results, scoring output, or
execution directives.

## Warning and Diagnostics Policy

- Forbidden fields are surfaced only through safe warnings and diagnostics.
- Forbidden fields are not copied into dashboard row fields.
- Disabled candidates are represented with `selectable=false`.
- Duplicate candidates are represented as review warnings and are not
  selectable.
- Insufficient data is represented as a review warning, not as advice.
- Empty watchlists are represented as a safe empty state.
- Invalid symbols are represented as safe warnings.

## Contract Reuse

- MS-09.01 candidate input contract is reused through source result candidate
  statuses and forbidden label policy.
- MS-09.02 watchlist model is reused through validated watchlist item and
  collection results.
- MS-09.03 manual/local source adapter is reused as the in-memory source result
  input.
- MS-09.04 fixtures are reused to build fixture-based dashboard preflights.

## Fixture-Based Preflight

`build_dashboard_preflight_from_fixture` and
`build_all_fixture_dashboard_preflights` use only in-memory fixture builders and
the MS-09.03 source adapter. No fixture file loader, file read, DB read, API
read, local snapshot read, scoring, or UI rendering is performed.

## Display Policies

- Forbidden field sanitization: confirmed through the sanitized fixture.
- Forbidden label/action prevention: confirmed by badge, row, status, and hint
  checks.
- Duplicate display policy: duplicate rows become non-selectable review rows.
- Disabled display policy: disabled rows become `selectable=false`.
- Insufficient data display policy: data shortage becomes review warnings and
  the `insufficient_data` safety badge.
- Empty watchlist display policy: empty input becomes a safe empty state with
  `review_empty_watchlist_input`.
