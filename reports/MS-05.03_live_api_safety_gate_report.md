# MS-05.03 Live API Safety Gate report

## 수행 내용

- 실제 전송 경로와 분리된 metadata-only `LiveApiSafetyGate`를 구현했습니다.
- 공식 확인된 read-only endpoint allowlist와 write/order denylist를 적용했습니다.
- accountSeq 필요, unknown, pending endpoint를 fail-closed 방식으로 차단했습니다.

## Live API Safety Gate 구조 요약

- `TossEndpointMetadata`: method, path, category, auth/account 요구 여부
- `LiveApiSafetyGate.evaluate()`: result 기반 fail-closed 평가
- `LiveApiSafetyGate.check_or_raise()`: 차단 시 `LiveApiSafetyError`
- `LiveApiSafetyGate.evaluate_settings()`: 기존 `Settings` 안전 flag 재사용
- network client, token provider, authenticated context, request send와 연결하지 않음

## read-only allowlist 요약

- `GET /api/v1/stocks`
- `GET /api/v1/stocks/{symbol}/warnings`
- `GET /api/v1/prices`
- `GET /api/v1/candles`
- `GET /api/v1/exchange-rate`

Allowlist는 `ALLOW_LIVE_API=true`, `ALLOW_REAL_ORDER=false` 조건에서 dry-run
metadata 평가만 허용합니다. `orderbook`, `trades`, `price-limits`는 pending으로
분리하여 기본 차단합니다.

## write/order denylist 요약

- `POST /api/v1/orders`
- `POST /api/v1/orders/{orderId}/modify`
- `POST /api/v1/orders/{orderId}/cancel`
- `order`, `write`, `trading`, `mutation` category
- `POST`, `PUT`, `PATCH`, `DELETE` 후보

주문 API는 설정과 무관하게 critical 위험으로 차단합니다.

## accountSeq required API 기본 차단 정책 요약

accountSeq 필요 endpoint는 read-only라도 이번 단계에서 기본 차단합니다. 실제
계좌 범위 API는 별도 Micro Stage와 사용자 승인이 필요합니다.

## 설정 안전조건 요약

- `ALLOW_LIVE_API=false`: live 후보 차단
- `ALLOW_REAL_ORDER=true`: critical 위험으로 차단
- `DRY_RUN_ONLY=true` + send 후보: 차단
- `DRY_RUN_ONLY=true` + allowlist metadata 평가: allowed decision
