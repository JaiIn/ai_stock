SQLite 파일에 저장하기 전에, 파일 경로·Git 제외·외부 동작 차단·DB 생성 금지

schema initialization 또는 repository file connection을 실행하지 않았습니다.

## Local SQLite DB file preflight 설계 요약

`LocalSnapshotDbFilePreflightPlan`은 `frozen=True`, `slots=True` dataclass입니다.
다음 단계가 별도로 승인되기 전까지 모든 실행성 동작을 비활성화합니다.

- stage: `MS-06.05`
- storage mode: `local_sqlite_file_preflight`
- planned DB relative path: `data/local/ai_stock.sqlite3`
- DB file creation allowed this stage: `false`
- actual DB file created: `false`
- data directory creation allowed: `false`
- schema initialization allowed: `false`
- repository file connection allowed: `false`
- API call allowed: `false`
- OAuth token endpoint allowed: `false`
- live smoke execution allowed: `false`
- env file read allowed: `false`
- credential required: `false`
- accountSeq allowed: `false`
- real order related call allowed: `false`

builder와 validator는 client, config, repository, `sqlite3`, network 모듈을
import하지 않습니다. 파일 열기·디렉터리 생성·SQLite 연결도 수행하지 않습니다.

## Planned DB path

계획 경로는 repo-relative 고정값
`data/local/ai_stock.sqlite3`입니다.

validator는 다음 경로를 거부합니다.

- 절대 경로 또는 `..`가 포함된 repo 외부 경로
- `.env.local`과 `.env`
- credential, secret, token 파일로 판단되는 경로
- 고정 계획값과 다른 경로

## DB 파일 생성 금지 및 실제 생성 결과

- `db_file_creation_allowed_this_stage=false`
- `data_directory_creation_allowed=false`
- `schema_initialization_allowed=false`
- `repository_file_connection_allowed=false`
- `actual_db_file_created=false`
- `data/` 존재: 없음
- `data/local/` 존재: 없음
- `*.sqlite`, `*.sqlite3`, `*.db` 생성: 없음

validator는 실제 파일 존재 여부를 caller-supplied boolean observation으로만
받습니다. 자체 filesystem I/O는 수행하지 않으며 파일이 존재한다고 입력되면
preflight를 거부합니다.

## Required Git ignore pattern

immutable plan에 다음 필수 정책을 고정했습니다.

- `data/`
- `*.sqlite`
- `*.sqlite3`
- `*.db`

현재 `.gitignore`에는 전역 `*.sqlite`, `*.sqlite3`, `*.db` 규칙과
`data/local/` 확장자별 규칙이 있습니다. 정확한 `data/` 규칙은 없습니다.
따라서 validator에 현재 `.gitignore` 규칙을 입력하면
`missing_ignore_patterns=("data/",)`이며 file DB stage readiness는 `false`입니다.

이번 허용 변경 범위에는 `.gitignore`가 포함되지 않으므로 수정하지 않았습니다.
향후 실제 파일 DB 생성 전 별도 승인 단계에서 정확한 `data/` 제외 규칙을
추가하고 preflight를 다시 통과해야 합니다.

## Validation helper 요약

`validate_local_snapshot_db_file_preflight()`는 다음 항목을 검증합니다.

- stage와 storage mode 고정값
- 계획 경로의 repo-relative·고정값 여부
- credential 경로가 아닌지 여부
- DB·디렉터리·schema·repository 실행 flag가 모두 `false`인지 여부
- 실제 DB 파일이 없다는 caller observation
- 필수 ignore pattern 누락 여부
- API/OAuth/live smoke/env/credential/accountSeq/order flag가 모두 `false`인지 여부

검증 결과는 immutable `LocalSnapshotDbFilePreflightResult`로 반환하며,
MS-06.05에서는 `ready_for_file_db_creation=false`를 항상 유지합니다.
