## 공식 OpenAPI 재확인 결과

- Source: `https://openapi.tossinvest.com/openapi-docs/latest/openapi.json`
- OpenAPI version: `3.1.0`
- API version: `1.1.5`
- Endpoint: `GET /api/v1/prices`
- operationId: `getPrices`
- Security: OAuth2 Client Credentials
- accountSeq required: false
- Read-only: true
- Rate Limits Group: `MARKET_DATA`

## Request query schema

- `symbols`: required
- Cardinality: 1 to 200
- Encoding: comma-separated
- Allowed symbol characters: English letters, digits, `.`, `-`
- Single-symbol request: supported
- Next proposed live candidate: `GET /api/v1/prices?symbols=005930`
- Live execution in MS-05.12: not performed

## Response 200 schema

The root is the standard `{ "result": [...] }` envelope.
Response result 배열 schema를 확인했습니다.

- `symbol`: required string
- `lastPrice`: required decimal string, parsed as `Decimal`
- `currency`: required string; official values currently include KRW and USD
- `timestamp`: optional/nullable date-time
- `symbol`, `lastPrice`, `currency` required
- `timestamp` nullable/optional

The existing `PriceSnapshot.stock_code` and `PriceSnapshot.price` fields remain
unchanged for repository compatibility. Official aliases `symbol` and
`last_price` are exposed without changing storage callers.

## Error schema

`PriceError` safe parser를 추가했습니다. The Prices error parser retains only
documented safe metadata:

- `error.requestId`
- `error.code`
- `error.message`
- optional `error.data.field`
- optional `error.data.allowedValues`
- optional `error.data.constraint.min`
- optional `error.data.constraint.max`
- requestId, code, message, data.field, data.allowedValues,
  data.constraint.min/max만 안전 추출

Covered response categories:

- 400 `invalid-request`
- 404 common not-found response
- 429 `rate-limit-exceeded`
- 500 `internal-error`

Raw request bodies, raw response bodies, authorization headers, credentials,
and tokens are not retained by the model.

## Live API Safety Gate

- method: `GET`
- path: `/api/v1/prices`
- category: `market-data`
- requires auth: true
- requires accountSeq: false
- read-only allowlist: yes
- order/write/mutation category: no
- `requires_account_seq=true`: blocked

No change to the production safety allowlist was required. Tests now explicitly
cover the Prices account-free decision and fail-closed account-scoped metadata.
