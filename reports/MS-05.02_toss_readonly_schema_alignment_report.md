## getExchangeRate alignment 요약

- Request query는 optional `dateTime`만 허용합니다.
- 기존 `baseCurrency`와 `quoteCurrency` query 요구를 제거했습니다.
- Response의 `baseCurrency`, `quoteCurrency`, `rate`, `validFrom`, `validUntil`을 파싱합니다.
- `rate`는 `Decimal`로 변환합니다.
- 기존 저장 계층 호환을 위해 내부 `exchange_rate` field를 유지하고 `rate` property를 제공합니다.
- `valid_from`, `valid_until`은 optional로 처리합니다.
- 기존 로컬 ingestion fixture용 legacy mapping은 호환 경로로 유지했습니다.

## getCandles alignment 요약

- 공식 `result` object root를 검증합니다.
- `CandlePage`가 `candles` tuple과 optional `next_before`를 보존합니다.
- 기존 `Candle`의 Decimal OHLCV parsing은 유지했습니다.
- `nextBefore`가 있거나 없는 fake official payload를 모두 테스트합니다.

## 변경된 모델/파서/클라이언트 구조 요약

- `ExchangeRate`: 공식 환율 field와 validity 범위를 반영했습니다.
- `CandlePage`: candle 목록과 pagination cursor를 함께 반환합니다.
- `TossMarketInfoClient`: optional `dateTime` request만 생성하며 전송하지 않습니다.
- `TossMarketDataClient`: candle object root를 `CandlePage`로 파싱합니다.

## 기존 mock 가정과 변경된 점

- 환율 요청 통화쌍 query를 제거했습니다.
- 환율 응답의 `exchangeRate`/`dateTime` 가정을 공식 `rate`/`validFrom`/`validUntil`로 교체했습니다.
- candle response root의 list 가정을 object로 교체했습니다.
- 기존 DB schema와 repository는 변경하지 않았습니다.
