# MS-04.03 Paper Simulated Execution Service Report

## 수행 내용

- `PaperSimulatedExecutionService`를 추가해 검증된 `PaperOrder`를 명시적으로 전달된 모의 체결 가격으로 처리하도록 구성했습니다.
- 체결 결과는 `PaperExecutionResult`로 반환하며, 갱신된 `PaperPortfolio`, `PaperOrder`, `PaperTrade`, 현금 변화량, 보유 수량 변화량을 포함합니다.
- 기존 `PaperOrderValidationService`와 `assert_paper_trading_only()`를 재사용했습니다.
- `docs/19_DETAILED_MICRO_WBS.md`의 MS-04.03 정의를 모의투자 체결 서비스 범위로 보완했습니다.

## 모의 체결 규칙

- 입력은 `PaperPortfolio`, `PaperOrder`, 호출자가 직접 전달한 `simulated_execution_price`로 제한했습니다.
- `CREATED` 상태 주문만 체결 대상으로 허용합니다.
- 체결 성공 시 원본 주문을 직접 변경하지 않고 `updated_order`를 `FILLED` 상태로 반환합니다.
- 매수 체결은 `quantity * simulated_execution_price`를 모의 현금에서 차감하고, 보유 종목이 없으면 새 `PaperHolding`을 생성합니다.
- 기존 보유 종목 매수 시 보유 수량을 증가시키고 가중 평균단가를 최소 계산합니다.
- 매도 체결은 `quantity * simulated_execution_price`를 모의 현금에 더하고, 보유 수량을 감소시킵니다.
- 매도 후 보유 수량이 0이면 해당 `PaperHolding`을 결과 포트폴리오에서 제거합니다.
- 수수료와 세금은 이번 단계에서 0으로 간주했습니다.

## Safety guard 사용

- `assert_paper_trading_only()`를 체결 시작 시 호출합니다.
- `live`, `real`, `broker`, `production` 성격의 mode 또는 flag는 예외로 차단됩니다.

## 테스트 결과

- 매수 체결, 기존 보유 평균단가 갱신, 매도 체결, 전량 매도 후 holding 제거를 테스트했습니다.
- 현금 부족, 보유 수량 부족, terminal 상태 주문, non-positive 체결 가격, live flag 거절을 테스트했습니다.
- `socket.socket` patch로 테스트 중 실제 네트워크 호출이 발생하지 않음을 확인했습니다.

## 로컬 전용 및 금지 범위 확인

- 실제 Toss API 호출, OAuth Access Token 발급, Client ID/Secret/Token/accountSeq 사용은 구현하지 않았습니다.
- 실제 가격 조회/API 호출은 구현하지 않았고, 모의 체결 가격은 호출자가 직접 전달합니다.
- Toss client, httpx, OAuth, authenticated request context를 import하거나 생성하지 않았습니다.
- repository 또는 SQLite 저장 로직을 추가하지 않았습니다.
- 실제 주문 API, 실계좌 주문 전송, 실제 체결 조회, 계좌/자산/잔고 API를 구현하지 않았습니다.
- 실제 DB 파일 또는 `data/` 디렉터리를 생성하지 않았습니다.

## 후속 과제

- 수수료/세금 모델링 정책 확정
- 평균단가 반올림/정밀도 정책 확정
- 부분 체결, 취소, 거절, 체결 이력 저장 정책 확정
- 모의 포트폴리오 성과 계산은 별도 Micro Stage에서 다룰 예정
