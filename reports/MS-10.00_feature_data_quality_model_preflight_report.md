# MS-10.00 Feature/Data Quality Model Preflight Report

## Purpose

MS-10.00 adds a pure no-I/O feature/data quality contract on top of the MS-09
candidate, watchlist, manual/local source, fixture, and dashboard preflight
contracts. The new model expresses data usability, missing/insufficient data,
forbidden-field sanitization, and review needs. It does not create a score,
rank, recommendation, buy/sell/hold judgment, target price, expected return, or
profit probability.

## Feature/Data Quality Model Summary

The contract derives quality-only feature records and assessments from the
existing MS-09 dashboard preflight rows. It reuses in-memory fixture builders
and dashboard preflight builders, then maps each candidate state to a safe
quality status. The output is deterministic and side-effect free.

## Feature Quality Policy Summary

- Allowed feature names are fixed and descriptive.
- Forbidden feature names include score, rank, recommendation, action,
  target price, expected return, profit probability, credentials, account,
  order, holdings, balance, fills, and accountSeq-related fields.
- Allowed quality statuses are non-directive.
- Forbidden action/status labels reuse the MS-09 forbidden label policy.
- All external capability flags are required to remain false.
- Observation-only, no-scoring, and no-recommendation policies are explicit.

## Quality Policy Details

- Duplicate candidate: `quality_duplicate_candidate`, review required,
  future scoring usability false.
- Disabled candidate: `quality_disabled_candidate`, review required,
  future scoring usability false.
- Insufficient data: `quality_insufficient_data`, review required,
  future scoring usability false.
- Forbidden field sanitized: `quality_forbidden_field_sanitized`, diagnostics
  only, forbidden values are not copied into output models.
- Empty input: `quality_empty_input`, review required.
- Valid manual candidate: `quality_ok`, quality-only and non-recommendation.
