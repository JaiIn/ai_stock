# MS-10.02 Deterministic Feature Extraction Preflight Report

## Purpose

MS-10.02 adds a deterministic feature extraction preflight contract on top of
MS-10.00 feature quality assessments and MS-10.01 fixture/evaluator coverage.
The stage normalizes already in-memory quality outputs into future-scoring
candidate feature sets. It does not implement scoring, ranking,
recommendation, target price, expected return, profit probability, UI
integration, storage, file loading, DB access, Toss API, OAuth, OpenAI/LLM,
accountSeq, or order/account features.

## Deterministic Feature Extraction Preflight Summary

The new preflight layer converts feature quality assessments into immutable
feature extraction inputs and extracted feature sets. Extracted features are
copied only from MS-10.00 safe feature records, only when the candidate is
quality OK and usable for future scoring. Review, duplicate, disabled,
insufficient-data, invalid, forbidden-field sanitized, and empty-input cases
produce conservative non-ready extraction states.

## Feature Extraction Policy

- Allowed sources: `feature_quality_assessment`,
  `dashboard_preflight_assessment`, `watchlist_fixture_assessment`,
  `feature_quality_fixture_evaluation`, and
  `all_fixture_feature_quality_assessments`.
- Forbidden sources include file paths, feature files, watchlist files,
  database/sqlite/table sources, Toss API, OAuth, OpenAI/LLM, account,
  holdings, order, balance, fills, raw API response, raw DB row, credentials,
  tokens, and accountSeq.
- Required flags are inherited from MS-10.00 and remain false.
- Observation-only, no-scoring, no-ranking, no-recommendation, and deterministic
  in-memory-only policies are explicit.

## Extraction Policies

- Quality OK: maps to `extraction_ready` when all feature records are usable.
- Duplicate: maps to `extraction_duplicate_candidate`, review/blocked, no
  extracted features.
- Disabled: maps to `extraction_disabled_candidate`, review/blocked, no
  extracted features.
- Insufficient or missing data: maps to `extraction_missing_data`, no extracted
  features.
- Invalid candidate: maps to `extraction_invalid_candidate`, no extracted
  features.
- Forbidden field sanitized: maps to
  `extraction_forbidden_field_sanitized`; forbidden fields remain diagnostics
  only and raw forbidden values are not copied.
- Empty input: maps to `extraction_empty_input` with safe missing feature
  markers only.
