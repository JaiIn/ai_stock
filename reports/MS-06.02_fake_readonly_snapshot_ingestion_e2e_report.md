# MS-06.02 Fake Read-only Snapshot Ingestion E2E Report

## 수행 목적

MS-06.01에서 추가한 `ReadOnlySnapshotIngestionService`를 실제 Toss API나 live
client 없이 end-to-end로 실행했습니다. 고정된 fake provider가 이미 파싱된
domain model을 반환하고, 기존 `LocalDataPersistenceService`와 in-memory
SQLite를 통해 저장 및 repository 재조회까지 검증했습니다.

이번 smoke는 live read-only ingestion이 아닙니다.

## fake-only E2E smoke 설계 요약

`run_fake_readonly_snapshot_ingestion_smoke()`가 다음 고정 안전 입력으로 하나의
snapshot ingestion을 수행합니다.

- symbol: `005930`
- exchange-rate pair: `USD/KRW`
- candle interval: `1d`
- candle count: `1`
- adjusted: `true`

함수는 `create_connection(":memory:")`로 연결을 만들고
`LocalDataPersistenceService`와 `ReadOnlySnapshotIngestionService`를
조립합니다. 저장 후 같은 persistence service의 list 메서드로 각 모델을 다시
조회해 count, Decimal, timestamp 값을 검증한 뒤 연결을 닫습니다.

## 사용한 fake provider 요약

- `FakeStockInfoProvider`: 이미 파싱된 `StockInfo` 1개 반환
- `FakePriceSnapshotProvider`: 이미 파싱된 `PriceSnapshot` 1개 반환
- `FakeCandlePageProvider`: `Candle` 1개를 포함한 `CandlePage` 반환
- `FakeExchangeRateProvider`: 이미 파싱된 `ExchangeRate` 1개 반환

Toss client, OAuth provider, HTTP transport, 설정 로더는 import하지 않습니다.

## 문제점 또는 확인 필요 사항

최초 대상 테스트에서 Python 임시 디렉터리의 sandbox permission 오류가
발생했습니다. 실제 기능이나 SQLite 오류가 아니며, 디렉터리를 만들지 않는
실행 전후 DB 파일 상태 비교 방식으로 테스트를 정렬했습니다. 실패 실행에서
남은 작업공간 임시 디렉터리는 확인 후 제거했습니다.
