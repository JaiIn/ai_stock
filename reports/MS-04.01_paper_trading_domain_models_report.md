# MS-04.01 Paper Trading Domain Models Report

## 수행 내용

- 모의투자 전용 domain model을 `src/ai_stock/paper_trading/models.py`에 추가했다.
- 실주문 방지용 최소 safety guard를 `src/ai_stock/risk/paper_trading.py`에 추가했다.
- 실제 Toss API 호출, OAuth 토큰 발급, httpx client 생성, authenticated request context 생성은 구현하지 않았다.
- repository, SQLite schema, DB 파일, pyproject는 변경하지 않았다.

## 모델 구조

- `PaperPortfolio`: 모의 포트폴리오 식별자, 현금 잔고, 통화, 모의 보유 목록
- `PaperHolding`: 모의 보유 종목, 수량, 평균 단가, 통화
- `PaperOrder`: 모의 주문 요청 모델
- `PaperTrade`: 모의 체결 결과 모델
- `PaperOrderSide`: `BUY`, `SELL`
- `PaperOrderType`: `MARKET`, `LIMIT`
- `PaperOrderStatus`: `CREATED`, `ACCEPTED`, `CANCELLED`, `REJECTED`, `FILLED`

## 검증 규칙

- 금융 숫자는 `Decimal`만 허용한다.
- 수량과 가격은 0보다 커야 한다.
- 모의 포트폴리오 현금 잔고는 음수가 될 수 없다.
- 매수/매도, 주문 유형, 주문 상태는 enum으로 제한한다.
- 주문 상태 전이는 최소 안전 규칙을 따른다.

## 안전 정책

- `PaperTradingSafetyError`를 추가했다.
- `assert_paper_trading_only()`는 paper/mock/simulated 모드만 허용한다.
- live/real/broker/production 모드 또는 live-order intent flag가 들어오면 예외를 발생시킨다.

- Toss client, OAuth, httpx Client, authenticated request context, send 호출을 사용하지 않았다.
- 실제 주문 API, 실계좌 주문 전송, 계좌/자산/잔고/체결 API는 구현하지 않았다.
