# MS-03.03 Local Mock Ingestion Pipeline Report

## 수행 내용

- `LocalMockIngestionService`를 추가해 fake/mock payload를 기존 모델 객체로 변환하고 `LocalDataPersistenceService`로 저장하는 로컬 ingestion pipeline을 구현했다.
- 대상 데이터는 `StockInfo`, `PriceSnapshot`, `Candle`, `ExchangeRate`로 제한했다.
- 실제 Toss API 호출, OAuth 토큰 발급, client/request/auth context/transport 생성은 구현하지 않았다.

## ingestion 구조

- `ingest_stocks_payload(payload)`: fake stock payload를 `StockInfo`로 변환 후 upsert 저장한다.
- `ingest_price_snapshots_payload(payload)`: fake price payload를 `PriceSnapshot`으로 변환 후 append 저장한다.
- `ingest_candles_payload(stock_code, interval, payload)`: fake candle payload를 `Candle`로 변환 후 append 저장한다.
- `ingest_exchange_rates_payload(base_currency, quote_currency, payload)`: fake exchange-rate payload를 `ExchangeRate`로 변환 후 append 저장한다.
- `ingest_market_dataset(dataset)`: 위 payload들을 하나의 로컬 dataset 형태로 묶어 저장한다.

## parser 재사용 여부

- 기존 모델의 `from_mapping()` parser를 재사용했다.
- client instance, request definition, auth context, transport, send 호출은 사용하지 않았다.
- 추후 OpenAPI schema가 확정되면 parser 위치와 payload envelope 규칙 정리가 필요하다.

## 저장 흐름

1. fake payload 입력
2. 기존 model `from_mapping()` 호출
3. `LocalDataPersistenceService` 저장 메서드 호출
4. in-memory SQLite repository에 저장
5. persistence 조회 helper로 저장 결과 검증

## 로컬 전용 및 실제 호출 없음 확인

- 테스트는 `create_connection(":memory:")`만 사용했다.
- workspace에 `data/ai_stock.sqlite3`, `*.sqlite`, `*.sqlite3`, `*.db` 파일을 만들지 않았다.
- `socket.socket` patch로 ingestion 중 실제 네트워크 호출이 발생하지 않음을 확인했다.
- source 정적 검사로 API client, token provider, request context, httpx Client, send 호출을 사용하지 않음을 확인했다.

## 민감정보 저장 없음 확인

- 실제 Client ID, Client Secret, Access Token, accountSeq, 계좌번호를 요구하거나 저장하지 않는다.
- API Key / Token / Secret 저장 로직을 추가하지 않았다.
- 계좌/자산/주문/실주문/포트폴리오/잔고/체결 서비스는 구현하지 않았다.

## 추후 보완 사항

- 중복 payload 처리 정책과 retention policy는 후속 Micro Stage에서 확정한다.
- timestamp/date_time timezone 정규화 정책은 후속 Micro Stage에서 확정한다.
