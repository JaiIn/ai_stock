# MS-04.04 Paper Portfolio Valuation Service Report

## 수행 내용

- `PaperPortfolioValuationService`를 추가해 `PaperPortfolio`와 호출자가 직접 전달한 `simulated_current_prices`만으로 모의 포트폴리오 평가 결과를 계산했습니다.
- `PaperHoldingValuation`, `PaperPortfolioValuationResult`, `PaperPortfolioValuationError`를 추가했습니다.
- 기존 `assert_paper_trading_only()` safety guard를 재사용했습니다.
- `docs/19_DETAILED_MICRO_WBS.md`의 MS-04.04 정의를 모의투자 포트폴리오 평가 서비스 범위로 보완했습니다.

## 평가 규칙

- `cash_balance`는 원본 값을 그대로 평가 결과에 포함합니다.
- holding market value는 `quantity * current_price`로 계산합니다.
- holding cost basis는 `quantity * average_price`로 계산합니다.
- holding unrealized PnL은 `market_value - cost_basis`로 계산합니다.
- `total_holdings_value`, `total_portfolio_value`, `total_cost_basis`, `total_unrealized_pnl`을 Decimal 기반으로 합산합니다.
- cost basis가 0보다 크면 unrealized PnL rate를 계산하고, 0이면 `None`으로 처리해 0 나누기를 방지합니다.
- price map에 포함된 미보유 종목 가격은 이번 단계에서 무시합니다.

## 가격 검증

- 보유 종목의 current price가 누락되면 `PaperPortfolioValuationError`를 발생시킵니다.
- current price는 `Decimal`이어야 하며 0보다 커야 합니다.

## Safety guard 사용

- `value_portfolio()` 시작 시 `assert_paper_trading_only()`를 호출합니다.
- `live`, `real`, `broker`, `production` 성격의 mode 또는 flag는 예외로 차단됩니다.

## 테스트 결과

- cash-only portfolio 평가를 테스트했습니다.
- 단일 holding 평가와 복수 holding 평가를 테스트했습니다.
- 평가손익 양수/음수 케이스를 테스트했습니다.
- 가격 누락, 0 이하 가격, live flag 거절을 테스트했습니다.
- 원본 `PaperPortfolio`와 `PaperHolding`이 직접 변경되지 않음을 테스트했습니다.
- `socket.socket` patch로 테스트 중 실제 네트워크 호출이 발생하지 않음을 확인했습니다.

## 로컬 전용 및 금지 범위 확인

- 실제 Toss API 호출, 실제 시세/가격 API 호출, OAuth Access Token 발급을 구현하지 않았습니다.
- 실제 Client ID, Client Secret, Access Token, accountSeq를 사용하지 않았습니다.
- Toss client, httpx, OAuth, authenticated request context를 import하거나 생성하지 않았습니다.
- repository 또는 SQLite 저장 로직을 추가하지 않았습니다.
- 실제 주문 API, 실계좌 주문 전송, 실제 체결 조회, 계좌/자산/잔고 API를 구현하지 않았습니다.
- 실제 DB 파일 또는 `data/` 디렉터리를 생성하지 않았습니다.

## 후속 과제

- 수익률 반올림 및 표시 자릿수 정책 확정
- 가격 시점 및 평가 기준일 필드 도입 여부 검토
- 수수료/세금 반영 평가 정책 검토
- 모의 포트폴리오 평가 결과 저장 여부는 별도 Micro Stage에서 검토
