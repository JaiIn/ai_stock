# MS-02.03 Market Info Exchange Rate Mock 보고서

## 수행 범위

- `TossMarketInfoClient` request-only 인터페이스 추가
- `GET /api/v1/exchange-rate`와 문서화된 query parameter 정의
- `ExchangeRate` 최소 모델과 fake result object parsing 추가
- 환율 decimal string을 `Decimal`로 변환
- 실패형 `MockTransport` 기반 무전송 테스트 추가

## 안전성 확인

- 실제 Toss API 및 OAuth 호출 없음
- HTTP 전송 메서드 또는 계좌/자산/주문 API 구현 없음
- exchange-rate 요청은 accountSeq 없이 생성
- Authorization 원문과 raw headers를 예외 및 보고서에 저장하지 않음
- 테스트 값은 mock/fake 값만 사용

## OpenAPI schema 재검증 필요 사항

- 로컬 API 문서에는 응답 필드가 없어 fake fixture의 `exchangeRate`와 `dateTime`을 최소 가정함
- 실제 `result` wrapper 내부가 단일 object인지 추후 OpenAPI schema 재검증 필요
- 환율 기준 시각 필드가 `dateTime`, timestamp 또는 base date인지 재검증 필요
- provider/source 필드는 문서에 없어 모델에 임의 추가하지 않음
- 환율은 표시·참고용이며 실제 주문 환율과 다를 수 있음

## 문제점 또는 확인 필요 사항

- getExchangeRate 실제 endpoint path 재검증 필요
- `baseCurrency` / `quoteCurrency` / `dateTime` query parameter명 재검증 필요

- `date_time` / `base_date` / `timestamp` 중 공식 필드명 재검증 필요
