# MS-06.08 Live Snapshot Local DB File Preflight Report

## 수행 목적

다음 단계에서 live read-only snapshot을 기존 로컬 SQLite 파일에 안전하게
저장하기 전에 호출 범위, 기존 DB 공존, append/upsert, Git 보호, 기록 범위를
immutable no-I/O preflight로 고정했습니다.

이번 단계에서는 실제 API/OAuth 호출, live/fake smoke, 환경 파일 읽기,
SQLite 연결이나 쓰기를 수행하지 않았습니다.

## Live snapshot local DB file preflight 설계

`LiveSnapshotLocalDbPreflightPlan`은 frozen dataclass이며 다음 정책을
고정합니다.

- stage: `MS-06.08`
- planned DB relative path: `data/local/ai_stock.sqlite3`
- `db_file_write_allowed_this_stage=false`
- `actual_db_file_modified_this_stage=false`
- `existing_db_file_allowed=true`
- `db_file_git_tracked_allowed=false`
- `data_directory_git_tracked_allowed=false`
- `api_call_allowed_this_stage=false`
- `oauth_token_endpoint_allowed_this_stage=false`
- `live_smoke_execution_allowed_this_stage=false`
- `env_file_read_allowed_this_stage=false`
- `credential_required_this_stage=false`
- next-stage credential source: existing `.env.local` only
- `account_seq_allowed=false`
- `real_order_related_call_allowed=false`
- `stock_warnings_included=false`
- `stock_warnings_deferred=true`

validator는 filesystem이나 DB를 직접 조사하지 않고 caller-supplied
observation만 평가합니다. 기존 DB의 존재는 허용하지만 이번 단계 수정,
Git 추적, ignore 누락, 외부 동작 활성화, 호출 범위 변조는 차단합니다.

## 다음 단계 OAuth/API 호출 후보

아래 호출은 다음 단계에서 별도 승인된 경우에만 각각 최대 1회 후보입니다.
MS-06.08에서는 실행하지 않았습니다.

1. `POST /oauth2/token`
2. `GET /api/v1/stocks?symbols=005930`
3. `GET /api/v1/prices?symbols=005930`
4. `GET /api/v1/candles?symbol=005930&interval=1d&count=1&adjusted=true`
5. `GET /api/v1/exchange-rate?baseCurrency=USD&quoteCurrency=KRW`

- expected OAuth token endpoint calls: `1`
- expected business API calls: `4`
- expected total network calls: `5`
- candle `before` parameter: 사용하지 않음

각 business endpoint는 GET, authenticated, read-only,
`requires_account_seq=false`로 고정했습니다.

## Fake DB와 live DB 공존 정책

MS-06.07 fake DB가 존재할 수 있으며 MS-06.08은 해당 파일을 수정하지
않습니다. 다음 MS-06.09 후보는 기존 DB를 삭제하거나 overwrite하지 않고
같은 DB에 live snapshot을 추가 저장하는 방향으로만 계획합니다.

중복 또는 append 결과는 DB 내용 전문이 아니라 repository count 증가량,
최소 존재 여부, timestamp 기준 safe summary만 기록합니다.

## Append/upsert 정책

- schema initialization: idempotent 필수
- StockInfo: 기존 repository 동작에 따른 upsert
- PriceSnapshot: snapshot row insert
- Candle: snapshot row insert
- ExchangeRate: snapshot row insert
- 저장 후 검증: repository count delta 또는 최소 존재 및 timestamp
- raw response body 전문 저장: 금지
- credential, Access Token, Authorization header 저장: 금지
- DB 내용 dump: 금지
