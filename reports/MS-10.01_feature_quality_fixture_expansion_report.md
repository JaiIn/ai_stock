## Feature Quality Fixture Expansion Summary

The new fixture layer builds immutable fixture expectation records and
evaluates them against the existing MS-10.00 feature quality assessment
builders. It uses in-memory MS-09 watchlist fixtures and dashboard preflight
conversion only. No fixture file loader, feature file loader, DB read/write,
network/API call, or UI path is introduced.

## Expected Vs Actual Evaluation Policy

The evaluator compares actual quality statuses, needs-review counts,
future-scoring usability counts, blocked reason keywords, warning keywords,
diagnostic keywords, required false flags, validator results, and forbidden
keyword absence. Mismatches return failures without throwing unless the source
fixture name itself is unknown.

## Fixture Quality Policies

- Duplicate fixture: produces `quality_duplicate_candidate`, needs review, and
  is not usable for future scoring.
- Disabled fixture: produces `quality_disabled_candidate`, needs review, and is
  not usable for future scoring.
- Insufficient data fixture: produces `quality_insufficient_data`, needs
  review, and is not usable for future scoring.
- Forbidden field sanitized fixture: produces
  `quality_forbidden_field_sanitized`; forbidden fields remain in diagnostics
  only and raw forbidden values are not copied into evaluator output.
- Empty input fixture: produces `quality_empty_input` as a safe review state.
- All fixture matrix: evaluates all six MS-09.04 fixture sources as one matrix.
